# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Branch context

This is `native-rendering-version` — a complete rewrite of the photo-manager UI from Vue/Tauri to pure Rust with SDL3 native rendering. The rewrite is in very early stages: the backend domain logic is complete and stable, the SDL3 rendering layer is a skeleton (`create_app()` is empty). Old Vue/TypeScript source files still exist in `src/` but are dead code being replaced.

## Commands

```bash
# Build
cargo build

# Run (logs to photo_manager.log in cwd; previous log is cleared on each run)
cargo run

# To add a new migration:
diesel migration generate <name>
diesel migration run
```

SDL3 is built from source via the `build-from-source-static` feature — first build takes several minutes.

## Architecture

This is a **pure Rust desktop app**: SDL3 for rendering + Tokio async runtime + SQLite via Diesel.

### Entry point

`src/main.rs` initializes logging, then calls `create_app()` from `src/app.rs`. `create_app()` is the SDL3 window/event loop entry point — currently an empty stub.

### Module layout

```
src/                  # Rust entry points and shared types
  main.rs             # Tokio main, logger setup
  app.rs              # create_app() — SDL3 window + event loop (skeleton)
  models.rs           # Diesel model structs (Photo, Tag, Person, Place, etc.)
  schema.rs           # Auto-generated by Diesel — do not edit by hand

src-tauri/src/        # Backend domain logic (stable, not yet moved to src/)
  app/                # DB init, photo scanning, EXIF extraction
    mod.rs            # Global state, row_to_vec/vec_to_row helpers
    search.rs         # Search query parser and executor
  photos/             # Photo metadata mutations, RAW/video handling, grouping
  people/             # Person and category management
  places/             # Geolocation, layers, polygon shapes
  tags/               # Tag graph (prerequisites, corequisites, incompatibilities)
  settings/           # Theme and color palette

migrations/           # Diesel migrations (embedded at compile time via MIGRATIONS const)
```

### Global state

Three `lazy_static` Tokio `Mutex`es live in `src-tauri/src/app/mod.rs`:

- `DB` — the SQLite connection, set during initialization; must be set before any query
- `OPEN_FOLDER` — the active photo library folder path
- `THUMBNAIL_DIR` — the thumbnail cache path (under `app_data_dir`)

Each domain module also has its own in-memory cache (e.g. `PHOTOS`, `TAGS`, `PEOPLE`, `PLACES`, `LAYERS`) loaded during initialization to avoid repeated DB queries.

### Database

SQLite managed by Diesel with async via `diesel-async`'s `SyncConnectionWrapper`. Migrations are embedded at compile time with `embed_migrations!("migrations")` and run automatically on startup.

Multi-valued fields (tags, people on a photo) are stored as comma-separated strings. Use `row_to_vec(option_string)` to parse and `vec_to_row(vec)` to serialize — both defined in `src-tauri/src/app/mod.rs`.

### SDL3 dependencies

- `sdl3 = "0.18.4"` with `build-from-source-static` and `ttf` features
- `sdl3-ttf-sys = "0.6"` — font rendering (no harfbuzz)

### Design system

See `DESIGN.md` for the full spec. The theme is "The Observatory" — near-monochrome dark with a muted slate blue accent:

- Primary accent: `oklch(65% 0.14 245)` ≈ `#5f87ac`
- Background: `oklch(9% 0.003 245)` (near-black)
- Surface: `oklch(14% 0.004 245)`
- Text primary: `oklch(94% 0.006 245)`
- Text secondary: `oklch(58% 0.006 245)`
- Base spacing: 4px grid; corner radius 4px (sm) / 8px (md)
- Typography: Roboto, 20px/700w headlines, 14px/400w body

SDL3 rendering should use these values directly as RGB approximations — the old CSS custom properties are gone with the Vue frontend.
